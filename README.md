# Manik Bot

Telegram-бот для записи клиентов на маникюр. Клиент может посмотреть услуги,
выбрать свободное время, записаться, отменить или перенести запись. Мастер
управляет услугами, расписанием и активными записями через админское меню.

## Возможности

- клиентское меню: услуги, моя запись, помощь;
- админское меню: услуги, расписание, записи, помощь;
- добавление и отключение услуг;
- добавление, просмотр и закрытие свободных слотов;
- самостоятельная запись клиента на свободный слот;
- отмена и перенос записи клиентом;
- отмена и перенос записи мастером;
- уведомления мастеру о новых записях;
- напоминания клиентам за сутки до записи.

## Конфигурация

Скопируйте пример настроек и заполните значения:

```bash
cp .env.example .env
```

Основные переменные:

- `BOT_TOKEN` - токен Telegram-бота;
- `ADMIN_IDS` - Telegram ID мастеров через запятую;
- `DATABASE_URL` - адрес PostgreSQL;
- `TIMEZONE` - часовой пояс, например `Europe/Moscow`.

## Локальная разработка

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
pytest
ruff check .
mypy
```

## Запуск через Docker Compose

Запустите PostgreSQL и бота:

```bash
docker compose up --build
```

При запуске контейнер бота автоматически применяет миграции Alembic.

Остановить контейнеры:

```bash
docker compose down
```

Остановить контейнеры и удалить данные PostgreSQL:

```bash
docker compose down -v
```

## Миграции

Для локального запуска без Docker Compose перед первым стартом примените миграции:

```bash
alembic upgrade head
```

Если база запущена через Docker Compose, команду можно выполнить из локального
окружения с корректным `DATABASE_URL` в `.env`.

## Команды бота

- `/start` - открыть главное меню;
- `/help` - показать помощь.

Клиенты используют кнопки меню для просмотра услуг и управления своей записью.
Мастер определяется по `ADMIN_IDS` и получает отдельное меню управления.
